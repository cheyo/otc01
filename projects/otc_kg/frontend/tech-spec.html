OTC / O2C-R2R 应用技术方案详解

当前系统已经从早期 OTC 风险原型，推进到标准版 O2C（订单到收款）+ R2R（记录到报告）重构阶段，并已落地 MySQL v2 与 Neo4j v2。

1. 已完成的阶段

  • 标准版 schema：schema_v2_o2c_r2r.sql
  • 标准版造数：generator_v2/scripts/generate_sample_data_v2.py
  • 标准版导入：load_v2.sql，数据库 otc_kg_v2
  • 标准版图谱：neo4j_v2/export_mysql_to_neo4j_v2.py + load_v2.cypher

6. 当前 v2 图谱关系

  • CustomerAccount → USES_BANK_ACCOUNT → BankAccount
  • CustomerAccount → PLACED_ORDER → SalesOrder
  • SalesOrder → BILLED_AS → Invoice
  • CustomerAccount → MADE_PAYMENT → CashReceipt
  • CashReceipt → APPLIED_TO → Invoice
  • Invoice → ACCOUNTED_AS → SubledgerEntry
  • SubledgerEntry → POSTED_TO → JournalEntry
  • JournalEntry → HITS_ACCOUNT → GLAccount
  • GLAccount → BALANCES_TO → GLBalance
